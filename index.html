História da Web Voltar ao início

Computadores

O primeiro boom tecnológico que ocasionou na criação dos primeiros computadores teve início a partir da Segunda Grande Guerra Mundial.

Dessa forma, podemos dividir a história dos computadores em:

  • Primeira Geração (1940-1956):
    • Uso de válvulas e componentes mecânicos.
    • Exemplos: ENIAC, Mark I e Colossus.

  • Segunda Geração (1956-1963):
    • Introdução do transistor;
    • Desenvolvimento da linguagem assembly.
    • Exemplos:IBM 7090, IBM 1401.

  • Terceira Geração (1964-1971):
    • Surgimento dos circuitos integrados;
    • Desenvolvimento de sistemas operacionais.
    • Exemplos: IBM System/360, DEC PDP-8.

  • Quarta Geração (1971-1989):
    • Introdução dos microprocessadores;
    • Surgimento dos computadores pessoais (PCs).
    • Exemplos: IBM PC, Apple Macintosh.

  • Quinta Geração (1989-atualmente):
    • Avanços em inteligência artificial e processamento paralelo;
    • Surgimento da Internet e da World Wide Web;
    • Miniaturização contínua e dispositivos móveis;
    • Exemplos: smartphones, supercomputadores.

Internet

Os telégrafos, que surgiram no século XIX, marcaram o início da comunicação instantânea à distância, transmitindo mensagens em código Morse através de fios elétricos.

A ARPANET, desenvolvida em 1969 pelo Departamento de Defesa dos EUA, foi a precursora da Internet, conectando quatro universidades e permitindo a troca de informações entre computadores distantes.

O protocolo TCP/IP, desenvolvido nos anos 1970, tornou-se o padrão de comunicação da Internet, permitindo a interconexão de redes heterogêneas.

Na década de 1980, a ARPANET cresceu rapidamente e o acesso à Internet tornou-se comercialmente disponível.

A World Wide Web, criada nos anos 1990 por Tim Berners-Lee no CERN, popularizou a Internet ao introduzir o conceito de hipertexto, navegadores e servidores web.

O boom da Internet no final da década de 1990 foi marcado pelo rápido crescimento de usuários, investimentos e inovações, impulsionando o surgimento de empresas ponto-com. Desde então, a Internet continua a evoluir, conectando bilhões de pessoas e dispositivos globalmente e dando origem a novas tecnologias e serviços, como redes sociais, streaming de vídeo, computação em nuvem e Internet das Coisas (IoT).

O que são Clients? Voltar ao início

Conceito de Client

Com a ajuda do diagrama acima, podemos identificar que, um client (cliente) é um dispositivo ou software que solicita e recebe serviços, recursos ou informações de outro dispositivo ou software chamado server (servidor). Os clients são componentes ativos em arquiteturas de rede cliente-servidor.


Exemplos de clients:
  • Navegadores da Web (browsers);

    • Um navegador da web funciona seguindo os passos a seguir:

    1. Requisição de URL: O usuário digita uma URL (Uniform Resource Locator) na barra de endereços do navegador ou clica em um link.
    2. Resolução de DNS: O navegador verifica o sistema de nomes de domínio (DNS) para traduzir o nome do domínio em um endereço IP correspondente.
    3. Solicitação HTTP: O navegador envia uma solicitação HTTP (Hypertext Transfer Protocol) ao servidor web associado ao endereço IP obtido, solicitando os recursos da página web.
    4. Resposta do Servidor: O servidor web recebe a solicitação, processa-a e envia de volta ao navegador uma resposta HTTP contendo os arquivos necessários para exibir a página web, como HTML, CSS, JavaScript e mídia.
    5. Renderização da Página: O navegador recebe os arquivos e os interpreta para renderizar a página web, exibindo-a na janela do navegador de acordo com as instruções de formatação fornecidas pelo HTML e CSS.
    6. Execução de Scripts: Se houver scripts (geralmente JavaScript) na página, o navegador os executa para adicionar interatividade e funcionalidades à página.
    7. Interação do Usuário: O usuário pode interagir com a página web clicando em links, preenchendo formulários, rolando a página, entre outras ações.
    8. Requisições Adicionais: Se a página contiver recursos externos, como imagens, vídeos ou scripts de terceiros, o navegador enviará solicitações adicionais ao servidor para obter esses recursos.
    9. Armazenamento em Cache: O navegador pode armazenar em cache alguns dos recursos da página para acelerar o carregamento subsequente da mesma página ou de páginas relacionadas.
    10. Atualizações e Eventos: O navegador continua a monitorar a página para atualizações, como alterações no conteúdo ou eventos do usuário, e responde a esses eventos de acordo.

  • Aplicações Web;

  • Dispositivos móveis.

Linguagens client-side

  • PHP;
  • Ruby;
  • JavaScript;
  • Java;
  • C#.

O que são Servers? Voltar ao início

Conceito de servers

Em tecnologia da informação e redes de computadores, um server é um dispositivo de hardware ou software que fornece serviços, recursos ou informações a outros dispositivos ou softwares chamados clients. Os servidores são projetados para atender solicitações de clientes e fornecer acesso a recursos compartilhados de forma eficiente e confiável.

Linguagens server-side

  • JavaScript;
  • TypeScript;
  • Dart;
  • WebAssembly (Wasm).

SOFTWARES E HARDWARES

Enquanto o hardware se refere aos componentes físicos do servidor, o software consiste nos programas e sistemas operacionais que permitem aos servidores executar serviços, aplicativos e cargas de trabalho específicos. Ambos desempenham papéis fundamentais na criação e operação de infraestruturas de servidor eficazes.

SOFTWARES:
Sistemas operacionais como o Windows Server, Ubuntu, CentOS, Fedora, etc;
Monitoramento (Nagios, Zabbix, etc);
Servidores web como Apache ou Nginx.
HARDWARES:
Armazenamento (disco rígido, SSDs);
Memória;
Processadores.

Como é uma estrutura básica de um HTML? Voltar ao início

A estrutura básica de um documento HTML (Hypertext Markup Language) consiste nos elementos fundamentais necessários para definir a estrutura e o conteúdo de uma página da web. Aqui está um exemplo da estrutura básica de um documento HTML:

  • !DOCTYPE html: Declara o tipo de documento e define que o documento é um HTML5.
  • html: É o elemento raiz que engloba todo o conteúdo da página.
  • head: Contém metadados sobre o documento, como o conjunto de caracteres, as tags de metadados (como meta), links para folhas de estilo (como link), scripts (como script), título da página (como title), entre outros.
  • title: Define o título da página, que é exibido na aba do navegador ou na janela do navegador.
  • body: Contém todo o conteúdo visível da página, como texto, imagens, vídeos, formulários, etc. Este é o local onde a maioria do conteúdo da página é colocado.

Atributos de Tags

Em HTML, os atributos de tags são usados para fornecer informações adicionais sobre os elementos HTML e especificar configurações ou comportamentos específicos. Eles são definidos dentro das tags e têm o formato nome="valor". Aqui estão alguns exemplos de atributos de tags HTML e suas descrições:

  • id: Define um identificador único para o elemento. É usado para selecionar o elemento via JavaScript ou CSS.
  • class: Define uma ou mais classes para o elemento. É usado para aplicar estilos CSS específicos ou selecionar o elemento via JavaScript.
  • src: Especifica o caminho do arquivo de origem para elementos como imagens, áudio ou vídeo.
  • href: Especifica o URL de destino para links (a) e âncoras.
  • style: Define estilos CSS inline para o elemento.
  • target: Especifica onde abrir o link quando clicado (por exemplo, em uma nova janela ou na mesma janela).
  • title: Fornecer um título ou dica de ferramenta para elementos. Quando o mouse passa sobre o elemento, o título é exibido.

Listas ordenadas e não ordenadas

No HTML, há dois tipos principais de listas: listas ordenadas (ol) e listas não ordenadas (ul)

As listas ordenadas (ol) são utilizadas para apresentar itens em uma sequência específica, onde cada item é numerado ou marcado de acordo com sua posição na lista.

Os itens são exibidos com um marcador numérico ou alfabético por padrão, mas podem ser personalizados usando o atributo type.

Os elementos de lista são definidos com a tag "li" (list item ou item de lista).


Já as listas não ordenadas são utilizadas para apresentar itens em uma sequência sem uma ordem específica, onde cada item é marcado com um símbolo de bullet por padrão.

Os elementos de lista são definidos com a tag "li", assim como nas listas ordenadas.

As listas não ordenadas são frequentemente utilizadas para representar menus de navegação, listas de itens ou qualquer conjunto de informações onde a ordem não é relevante.
